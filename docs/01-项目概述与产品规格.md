## 1. 产品概述

### 1.1 一句话描述

用户通过微信小程序扫码付款，租赁外骨骼设备进行步行辅助/康复训练，使用结束后自动扣费。

### 1.2 用户画像

| 用户类型 | 使用场景 | 租赁方式 |
|---------|---------|---------|
| **康复患者** | 术后步态训练 | 按次/按时租赁 |
| **户外爱好者** | 长途徒步辅助 | 单次租赁 |
| **体验用户** | 首次尝鲜 | 按分钟计费，用完即止 |

### 1.3 核心差异化（vs 共享充电宝）

| 维度 | 共享充电宝 | 外骨骼租赁 |
|------|----------|-----------|
| 鉴权方式 | 扫码即用 | 扫码 + 支付 + BLE 授权令牌 |
| 使用过程 | 无交互 | 实时控制（模式/等级）、运动数据反馈 |
| 结束方式 | 归还到机柜 | 主动停止 + 物理关机 |
| 数据留存 | 仅扣费记录 | 完整运动报告（步数、时长、热量） |
| 安全隐患 | 无 | 跌倒检测自动急停 |

│  │                        📍  │  │

│  │  📍 最近设备点               │  │  ← 附近设备点卡片
│  │  CDC康复中心 · 距您 320m    │  │     点击跳转设备详情
│  │  可租: 3台 · ¥30/小时起     │  │
│  └────────────────────────────┘  │
│                                  │
│       ┌──────────────┐           │
│       │   📷 扫码租赁  │           │  ← 核心 CTA 按钮
│       └──────────────┘           │     点击拉起扫码
│                                  │
│  使用须知 | 客服电话              │  ← 底部辅助入口
└──────────────────────────────────┘
```

**功能要点：**
- 进入首页请求定位权限，地图中心移到用户位置
- 设备点标记从服务端 API 拉取（含经纬度、可用数量、最低价格）
- 点击标记弹出设备点卡片（设备点名称、距离、可租数量、起租价）
- 扫码按钮为核心 CTA，使用 `uni.scanCode({ onlyFromCamera: true })` 仅允许相机扫码
- 扫码结果解析为设备 SN → 跳转确认使用页
- 无定位权限时，地图默认显示北京中心，设备列表按距离排序

**扫码流程：**

```
点击扫码 → 相机权限检查 → 扫码识别 →
  ├── 识别到外骨骼设备二维码 → 跳转确认使用页（携带设备SN）
  ├── 识别到非外骨骼二维码 → Toast "请扫描外骨骼设备二维码"
  └── 用户取消 → 回到首页
```

## 4. 全局交互规范

### 4.1 加载状态

```
       ◌ ◌ ◌         ← 三点呼吸动画（类似 iOS 活动指示器）
                     颜色跟随当前页面主色调
    加载中...
```

### 4.2 Toast 反馈

| 类型 | 图标色 | 位置 | 时长 | 示例 |
|------|-------|------|------|------|
| 成功 | 绿色 | 顶部弹出 | 2 秒 | "模式切换成功" |
| 失败 | 红色 | 顶部弹出 | 3 秒 | "指令发送失败" |
| 警告 | 黄色 | 居中弹出 | — | "请先停止运行再切换模式" |
| 信息 | 蓝色 | 顶部弹出 | — | "正在连接设备..." |

### 4.3 空状态

```
       📭         ← 空状态插画
   暂无使用记录
   快去扫码体验吧

  ┌──────────┐
  │  扫码租赁  │    ← 引导按钮
  └──────────┘
```

### 4.4 网络异常

```
       🌐
   网络连接异常
   请检查网络后重试

  ┌──────────┐
  │  重新加载  │
  └──────────┘
```

```

  └──────────┘
