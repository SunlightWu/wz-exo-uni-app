## 1. 系统整体架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   uni-app 小程序  │     │   云端服务端      │     │   外骨骼设备      │
│                 │     │                 │     │                 │
│ • 扫码租赁       │────▶│ • 订单管理        │     │ • BLE 通信       │
│ • 付款（微信支付） │◀────│ • 支付对接        │     │ • 固件控制       │
│ • BLE 设备连接    │◀───▶│ • 设备状态同步     │◀───▶│ • 传感器数据     │
│ • 运动数据展示    │     │ • 会话记录        │     │ • 故障上报       │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

**核心变化（相对现有 Flutter APP）：**
- 小程序通过微信支付完成付费鉴权，而非 APP 内无鉴权直连
- 服务端作为中间层管理设备租赁生命周期（开始/结束会话）
- 设备控制指令依然走 BLE 直连，但需服务端下发授权令牌
- 运动数据通过小程序 BLE 直读，同时上报服务端存档

## 2. BLE 连接协议（设备直连层）

> ⚠️ **本节的协议定义是所有文档的唯一权威来源**，开发实施请严格依据此处定义。

### 2.1 UUID 定义

```
GATT 主服务：    0000FFF0-0000-1000-8000-00805F9B34FB
命令通道(Write)： 0000FFF1-0000-1000-8000-00805F9B34FB   → 发送命令
状态通道(Notify)：0000FFF2-0000-1000-8000-00805F9B34FB   → 50Hz 状态数据 + ACK
参数通道(Write)： 0000FFF3-0000-1000-8000-00805F9B34FB   → 参数读写
故障通道(Notify)：0000FFF4-0000-1000-8000-00805F9B34FB   → 故障变化上报
实时通道(Notify)：0000FFF5-0000-1000-8000-00805F9B34FB   → 10Hz 传感器数据
日志通道(Notify)：0000FFF6-0000-1000-8000-00805F9B34FB   → 日志流
扫描过滤 UUID：   777a6b6a-3230-3236-6878-79717165786f   → 广播过滤
```

### 2.2 命令编码表

| 命令 | Code | Payload 格式 | 说明 |
|------|------|-------------|------|
| hello | 0x01 | `[0x01]` | 握手 |
| setUserProfile | 0x02 | 11 字节: height(f32 LE) + weight(f32 LE) + age(u8) + gender(u8) | 用户参数 |
| calibrateZero | 0x03 | `[0x03]` | 零点校准 |
| setMode | 0x04 | `[0x04, mode]` | mode: 0=transparent, 1=assist, 2=training |
| setAssistLevel | 0x05 | `[0x05, level]` | level: 1-10 |
| setImpedanceLevel | 0x06 | `[0x06, level]` | level: 1-10 |
| start | 0x07 | `[0x07]` | 开始运行 |
| stop | 0x08 | `[0x08]` | 停止运行 |
| clearFault | 0x09 | `[0x09]` | 清除故障 |
| disconnect | 0x0A | `[0x0A]` | 断开连接 |

#### 编码实现

```javascript
// ── 命令编码 ──
export function encodeCommand(code, payload = {}) {
  switch (code) {
    case CMD.hello:
    case CMD.calibrateZero:
    case CMD.start:
    case CMD.stop:
    case CMD.clearFault:
    case CMD.disconnect:
      return new Uint8Array([code]).buffer;

    case CMD.setUserProfile: {
      const buf = new ArrayBuffer(11);
      const v = new DataView(buf);
      v.setFloat32(0, payload.heightCm || 170, true);
      v.setFloat32(4, payload.weightKg || 70, true);
      v.setUint8(8, payload.age || 0);
      v.setUint8(9, payload.gender === 'male' ? 1 : 2);
      return buf;
    }

    case CMD.setMode: {
      const modes = { transparent: 0, assist: 1, training: 2 };
      return new Uint8Array([0x04, modes[payload.mode] ?? 0]).buffer;
    }

    case CMD.setAssistLevel:
    case CMD.setImpedanceLevel: {
      const lv = Math.max(1, Math.min(10, payload.level || 1));
      return new Uint8Array([code, lv]).buffer;
    }

    default:
      return null;
  }
}
```

### 2.3 数据上报格式

#### FFF2 状态包（30 字节，50Hz）

| 偏移 | 长度 | 类型 | 字段 | 说明 |
|------|------|------|------|------|
| 0 | 4 | uint32 LE | 时间戳 | ms |
| 4 | 1 | uint8 | 状态码 | 设备内部状态 |
| 5 | 1 | uint8 | 模式 | 0=transparent, 1=assist, 2=training |
| 6 | 1 | uint8 | 运行标志 | 0=停止, 1=运行中 |
| 7 | 1 | uint8 | 电池 | 0-100% |
| 8 | 4 | uint32 LE | 故障码 | 位掩码（见下） |
| 12 | 2 | int16 LE | 左髋角度 | 厘度 → 度（÷100） |
| 14 | 2 | int16 LE | 右髋角度 | 厘度 → 度（÷100） |
| 16 | 2 | int16 LE | 左力矩 | mN·m → N·m（÷1000） |
| 18 | 2 | int16 LE | 右力矩 | mN·m → N·m（÷1000） |
| 20 | 1 | uint8 | 助力等级 | 1-10 |
| 21 | 1 | uint8 | 阻抗等级 | 1-10 |
| 22 | 4 | uint32 LE | 步数 | 累计步数 |
| 26 | 4 | uint32 LE | 运行时长 | 秒 |

#### FFF5 传感器包（60 字节，10Hz）

| 偏移 | 长度 | 类型 | 字段 |
|------|------|------|------|
| 0 | 4 | uint32 LE | 时间戳 |
| 4 | 4 | float32 LE | roll 角度（度） |
| 8 | 4 | float32 LE | pitch 角度（度） |
| 12 | 4 | float32 LE | yaw 角度（度） |
| 16 | 4 | float32 LE | 加速度 X |
| 20 | 4 | float32 LE | 加速度 Y |
| 24 | 4 | float32 LE | 加速度 Z |
| 40 | 4 | float32 LE | 海拔（m） |
| 44 | 4 | float32 LE | 气压温度（℃） |
| 48 | 4 | float32 LE | 电机 0 温度（℃） |
| 52 | 4 | float32 LE | 电机 1 温度（℃） |
| 56 | 1 | uint8 | 电池（%） |

> 偏移 28-39（12 字节）为保留字段，暂不解析。

#### FFF4 故障包（12 字节，仅故障变化时发送）

| 偏移 | 长度 | 类型 | 字段 |
|------|------|------|------|
| 0 | 4 | uint32 LE | 时间戳 |
| 4 | 4 | uint32 LE | 故障码（位掩码） |
| 8 | 1 | uint8 | 状态码 |

#### 故障码位掩码

| 掩码 | 描述 |
|------|------|
| `0x00000001` | IMU 离线 |
| `0x00000004` | 电机 0 离线 |
| `0x00000008` | 电机 1 离线 |
| `0x00000010` | 控制不允许 |
| `0x00000020` | 参数无效 |
| `0x00000040` | 通信超时 |
| `0x00000080` | 电机未使能 |
| `0x00000100` | 电池低电量 |
| `0x00000200` | 自检失败 |
| `0x00000400` | 过温 |
| `0x00000800` | 关节角度超限 |
| `0x00001000` | 力矩超限 |
| `0x00002000` | CAN 通信错误 |
| `0x00004000` | 控制周期超时 |

#### 解析实现

```javascript
// ── FFF2 状态包解析 ──
export function parseFFF2(buffer) {
  if (buffer.byteLength < 30) return null;
  const v = new DataView(buffer);
  return {
    timestampMs: v.getUint32(0, true),
    stateCode: v.getUint8(4),
    mode: ['transparent', 'assist', 'training'][v.getUint8(5)] || 'transparent',
    running: v.getUint8(6) !== 0,
    battery: v.getUint8(7),
    errorCode: v.getUint32(8, true),
    leftHipAngle: v.getInt16(12, true) / 100,
    rightHipAngle: v.getInt16(14, true) / 100,
    leftTorqueNm: v.getInt16(16, true) / 1000,
    rightTorqueNm: v.getInt16(18, true) / 1000,
    assistLevel: v.getUint8(20),
    impedanceLevel: v.getUint8(21),
    stepCount: v.getUint32(22, true),
    runtimeSeconds: v.getUint32(26, true),
  };
}

// ── FFF5 传感器包解析 ──
export function parseFFF5(buffer) {
  if (buffer.byteLength < 60) return null;
  const v = new DataView(buffer);
  return {
    rollDeg: v.getFloat32(4, true),
    pitchDeg: v.getFloat32(8, true),
    yawDeg: v.getFloat32(12, true),
    accX: v.getFloat32(16, true),
    accY: v.getFloat32(20, true),
    accZ: v.getFloat32(24, true),
    altitudeM: v.getFloat32(40, true),
    baroTempC: v.getFloat32(44, true),
    motor0TempC: v.getFloat32(48, true),
    motor1TempC: v.getFloat32(52, true),
    battery: v.getUint8(56),
  };
}

// ── FFF4 故障包解析 ──
export function parseFFF4(buffer) {
  if (buffer.byteLength < 12) return null;
  const v = new DataView(buffer);
  return {
    timestampMs: v.getUint32(0, true),
    errorCode: v.getUint32(4, true),
    stateCode: v.getUint8(8),
  };
}

// ── 故障码 → 中文描述 ──
export function faultCodeDescription(code) {
  if (code === 0) return '正常';
  const masks = [
    [0x00000001, 'IMU离线'],       [0x00000004, '电机0离线'],
    [0x00000008, '电机1离线'],     [0x00000010, '控制不允许'],
    [0x00000020, '参数无效'],       [0x00000040, '通信超时'],
    [0x00000080, '电机未使能'],     [0x00000100, '电池低电量'],
    [0x00000200, '自检失败'],       [0x00000400, '过温'],
    [0x00000800, '关节角度超限'],   [0x00001000, '力矩超限'],
    [0x00002000, 'CAN通信错误'],    [0x00004000, '控制周期超时'],
  ];
  return masks.filter(([m]) => code & m).map(([, s]) => s).join('、') || '未知故障';
}
```

### 2.4 ACK 应答机制

设备在 **FFF2 通道**回 1 字节应答码，与状态包共用同一通道：

| 应答码 | 含义 |
|--------|------|
| `0x00` | 成功 |
| `0x01` | 运行中忙 |
| `0x02` | 状态不允许 |
| `0x03` | 故障中 |
| `0x04` | 参数无效 |
| `0x05` | 电机未就绪 |
| `0x06` | 超时 |

> **重要约束**：固件不支持序列号，APP 端同一时间只允许一个待确认命令（互斥锁）。

```javascript
// ── ACK 互斥锁 ──
export class AckLock {
  constructor() {
    this._pending = null;
    this._resolve = null;
  }
  acquire() {
    if (this._pending) throw new Error('ACK 通道被占用');
    this._pending = new Promise((resolve) => { this._resolve = resolve; });
    return this._pending;
  }
  release() {
    if (this._resolve) this._resolve();
    this._pending = null;
    this._resolve = null;
  }
}

// ── 处理 FFF2 notify 中的 ACK 字节 ──
let pendingResolve = null, pendingReject = null, ackTimeout = null;

export function handleAckByte(byte) {
  if (!pendingResolve) return;
  clearTimeout(ackTimeout);
  const res = pendingResolve, rej = pendingReject;
  pendingResolve = null;
  pendingReject = null;
  const messages = {
    0x00: null, 0x01: '运行中忙', 0x02: '状态不允许',
    0x03: '故障中', 0x04: '参数无效', 0x05: '电机未就绪', 0x06: '超时',
  };
  const msg = messages[byte];
  if (msg) rej(new Error(msg));
  else res();
}
```

### 2.5 连接流程

```
扫描(过滤UUID 777a6b6a...) → 连接 → 发现服务(FFF0) →
发现6个特性(FFF1-FFF6) → 订阅Notify(FFF2/FFF4/FFF5/FFF6) →
发送hello(0x01) → 等待ACK → 发送setUserProfile(0x02) →
发送setMode(0x04) → 发送setAssistLevel/setImpedanceLevel →
发送start(0x07) → 进入运行态
```

### 2.6 数据限流策略

小程序 BLE notify 回调频率较高，直接 setData 会导致性能问题：

| 数据源 | 原始频率 | 限流后 | 方法 |
|--------|---------|--------|------|
| FFF2 状态数据 | 50Hz | 4Hz（250ms） | 时间戳比较 |
| FFF5 传感器数据 | 10Hz | 4Hz（250ms） | 时间戳比较 |

```javascript
export function createThrottle(minIntervalMs) {
  let last = 0;
  return () => {
    const now = Date.now();
    if (now - last < minIntervalMs) return false;
    last = now;
    return true;
  };
}
```

## 3. 小程序 BLE 适配方案

### 3.1 uni-app BLE API 能力映射

| Flutter API | uni-app API |
|-------------|-------------|
| startScan | `uni.startBluetoothDevicesDiscovery` + `uni.onBluetoothDeviceFound` |
| connect | `uni.createBLEConnection` |
| discoverServices | `uni.getBLEDeviceServices` |
| discoverCharacteristics | `uni.getBLEDeviceCharacteristics` |
| writeCommand | `uni.writeBLECharacteristicValue` |
| setNotifyValue | `uni.notifyBLECharacteristicValueChange` |
| onValueReceived | `uni.onBLECharacteristicValueChange` |
| disconnect | `uni.closeBLEConnection` |
| 初始化适配器 | `uni.openBluetoothAdapter` |
| 获取设备列表 | `uni.getBluetoothDevices` |
| 获取适配器状态 | `uni.getBluetoothAdapterState` |

### 3.2 uni-app BLE 关键约束（vs Flutter）

| 维度 | Flutter | uni-app |
|------|---------|---------|
| 写入模式 | `withoutResponse: false`（默认可靠写） | `writeType: 'write'`（强制回复写）。**注意：Android 默认 `writeNoResponse`，外骨骼协议必须用 `write`** |
| value 类型 | `Uint8List` | `ArrayBuffer`（用 `DataView` 读写） |
| 最大写入字节 | 无限制 | 建议每次不超 20 字节。外骨骼最长指令 11 字节，安全 |
| notify 数据 | `List<int>` 流 | `onBLECharacteristicValueChange` 回调，`value` 为 `ArrayBuffer` |
| 并行写入 | 支持 await 串行 | **必须串行执行，并行调用会写失败** |
| notify 后立即 write | 无问题 | **Android 部分机型 notify 后立即 write 会 10008 错误，需加 50ms 延时** |

### 3.3 控制页状态机

```
idle → scanning → connecting → configuring → ready → running → stopping → completed
                                    │           │
                                    └── fault ◄─┘
```

与 Flutter APP 的 `OutputCommandState`（idle → starting → running → stopping）保持一致。

```
│   └── 退款/押金退还

```

### 4.2 令牌机制

服务端生成的 JWT 令牌结构：

```json
{
  "sub": "user_openid",
  "device_sn": "WZ-EXO-S3-001",
  "iat": 1719234567,
  "exp": 1719238167,
  "lease_id": "L20240624001",
  "rate": 0.5,
  "daily_cap": 120
}
```

- `rate`: 每分钟单价（元），由服务端配置
- `daily_cap`: 每日封顶价（元），0 表示不封顶

**流程：** 小程序扫码确认使用 → 调服务端创建订单 → 押金授权/支付 → 服务端返回 JWT 令牌（含计费信息） → 小程序开始 BLE 连接。设备端不需要联网，授权和计费逻辑在服务端 + 小程序本地校验。

### 5.2 与现有 Flutter APP 的差异对照

| 维度 | 现有 Flutter APP | 小程序 |
|------|-----------------|--------|
| 用户鉴权 | 无 | 微信登录（openid） |
| 设备授权 | APP 直连即用 | 需服务端下发令牌 |
| 付费 | 无 | 微信支付扫码付费 |
| 使用限制 | 无时间限制 | 按实际时长计费，有每日封顶 |
| 设备发现 | BLE 扫描 | 扫码获取设备 SN + BLE 扫描 |
| 数据存储 | SQLite 本地 | 上传服务端 |
| BLE 协议 | **完全相同** | **完全相同** |
| 控制页 UI | Flutter | uni-app（条件编译 MP-WEIXIN） |

## 6. 核心风险与应对

| 风险 | 应对 |
|------|------|
| 小程序 BLE 连接不稳定 | 增加自动重连 + 用户手动重试；连接超时 10 秒 |
| 设备信号范围内多用户 | 扫码绑定设备 SN → BLE 扫描时按设备名称精确匹配 |
| 超时未结束租赁 | 服务端定时任务检测 → 按实际时长计费并自动结束 |
| 小程序切后台 BLE 断连 | `onHide` 时保持连接 + `onShow` 时检查并重连；已有令牌则恢复 |
| Android/iOS BLE 差异 | uni-app 已封装平台差异；异常 case 需要真机分别测试 |
| 用户退款场景 | 服务端按实际使用时长 × 单价计算已用费用，余额原路退回 |
